Help

Saving Proofs:

In order to save a proof for later, a user must be signed in to the application. The user can name their proof in the "name your proof" input bar, if the proof that the user has entered has a name in the input bar and is also imcomplete, the proof will then be saved for later so the user can finish it later. If a user enters a proof with a name that has already been entered before, the old proof will be updated to the new one.

Loading Prepository Problems:

Users can only load proofs from the problem repository if they are signed in. When loading a proof from the problem repository The "name your proof", "Premises", and "Conclusions" input bars are all locked to issue a user is solving a problem specifically from the problem repository. If a user wishes to start from scratch at any time they can click on the "restart proof checking from scratch" button to do so.

Loading Finished Prepository Problems:

When a user loads a finished repository problem, similar to loading repository problems, The "name your proof", "Premises", and "Conclusions" input bars are all locked. This is done because the purpose of loading problems from this selection is intended for user to view repository problems they have already done and to see how they did them.
